# CLAUDE.md

このファイルは、このリポジトリで Claude Code (claude.ai/code) が動作する際のガイダンスを提供します。

## プロジェクト概要

**gonda-complaint-resolution** は、岡崎車輌部品株式会社の近隣住民（ゴンダ様）からの車両騒音苦情に対応するための専用プロジェクトリポジトリです。

### プロジェクト目標

1. ゴンダ様からの苦情内容を詳細に確認・記録
2. 社内で対応方針を検討・決定
3. 騒音軽減の具体的方法を検討・実施
4. ゴンダ様への対応・連絡
5. 実施計画の立案と経営層の承認取得

## リポジトリ構造

### ディレクトリ構成

```
.
├── 00_inbox/          # 新規情報・タスクの一時格納
├── 01_research/       # 詳細ヒアリング・情報収集
├── 02_planning/       # 対応方針の検討・計画
├── 03_implementation/ # 改善策の実装
├── 04_followup/       # フォローアップ・検証
└── archive/           # 完了したドキュメント
```

### Obsidian Vault

このリポジトリは Obsidian Vault として機能し、プロジェクトの全体像を管理します。

- `.obsidian/` - Obsidian 設定ファイル
- Markdown ファイルで情報を記録・管理
- 他のノートとのリンクで関連情報を管理

## ドキュメンテーション標準

- **言語**: 日本語
- **形式**: Markdown
- **対象者**: プロジェクト参画者及び経営層

## プロジェクトフロー

### Phase 1: 詳細ヒアリング（01_research/）

- ゴンダ様との連絡・ヒアリング記録
- 苦情内容の詳細確認（時間帯、頻度、影響度など）
- 現状の運行ルート・スケジュール確認
- 関連法規・騒音規制の調査

### Phase 2: 社内検討（02_planning/）

- 社内会議の実施記録
- 対応方針の検討内容
- 複数の改善案の比較検討
- 経営層への提案資料作成

### Phase 3: 改善実施（03_implementation/）

- 決定した対応内容の実装計画
- 運行ルート・時間帯変更の具体案
- 関連部署への指示・通知
- 実施状況の記録

### Phase 4: フォローアップ（04_followup/）

- ゴンダ様への報告・説明記録
- 改善状況の定期確認
- 効果測定・検証
- 必要に応じた追加対応の検討

## GitHub Issues

GitHub Issues を通じて、進捗管理と課題追跡を行います。

- 各フェーズごとに Issue を作成
- ラベルで分類（research, planning, implementation, followup）
- 進捗を Issues の Checklist で管理

## 注意事項

- ゴンダ様との対応は丁寧・誠実に行うこと
- すべての対応記録をドキュメント化
- 法的観点（騒音規制等）の確認を忘れずに
- 実施前に経営層の承認を取得すること
- 定期的なフォローアップで信頼関係を構築
